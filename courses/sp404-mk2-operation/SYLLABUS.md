# MUS-40H Syllabus: Operating the SP-404 MK2

## Course description

A three-week intensive in **operating** the Roland SP-404 MK2, for the beginner who wants to understand and command the machine itself. This is not a beatmaking course (its sister, MUS-45H, is). It is a complete, hands-on tour of how the SP is built and how you drive it: navigation and the menu system, the full signal chain from input to output, the pads as a performance surface, recording and editing operations, the four-bus effects engine, the pattern sequencer, resampling, DJ mode, and connecting the machine to a computer over USB audio and MIDI. By the end, students can pick up an SP-404 MK2 cold, find any function, route any signal, and use every major subsystem with confidence.

This is an **applied operation course**: most of the time is spent driving the machine, every day, with the unit in your hands.

## Learning objectives

By the end of this course you will be able to:

1. **Navigate the entire interface** using the [VALUE] knob, [SHIFT], the mode buttons, and the system menu, and read the screen to know what every control is doing right now.
2. **Trace the machine's signal flow** from any input, through the pads and the four effect buses, to the outputs, and explain where any sound is at any moment.
3. **Manage projects, storage, and backup** so that work is never lost: know where work lives, and back it up and restore it.
4. **Connect and configure every input and output** (line, mic, phones, USB audio), set healthy levels, and choose mono or stereo.
5. **Play and perform on the pads** across all 10 banks using velocity, the three play modes, hold/remain, and the sub pad.
6. **Record a clean sample** from any source using level-setting, auto-trigger, INPUT FX, and skip-back, and **edit** it (trim, loop, mono, pitch) and manage pads (copy, move, delete).
7. **Operate the four-bus effects engine**: route pads to BUS 1/2, apply master effects on BUS 3/4, use INPUT FX, and reason about which bus affects what.
8. **Program the pattern sequencer** in both real-time and step (TR-REC) modes, with quantize and swing, and edit and chain patterns.
9. **Resample** the machine's own output to commit effects, glue sequences, and free up voices, and use it as the spine of a workflow.
10. **Connect the SP to the outside world**: operate DJ mode, use the SP as a USB audio interface into a DAW, sync over MIDI clock, control and be controlled by MIDI, and import/export samples and projects.

## Daily structure (~45–75 minutes)

| Block | Time | Activity |
|---|---|---|
| Power up & warm up | 5 min | Turn on, load your project, run the day's navigation drill from the Practice Menu |
| Orient | 3–5 min | Find on the machine the controls today's lecture will name, before you read about them |
| Lecture | 10–15 min | Read the lecture; do each small example at the machine as you read it |
| Operate | 20–40 min | The day's machine task: navigate, route, record, sequence, connect |
| Review | 5 min | Answer the self-check from memory; note what to revisit |

**Mastery threshold: 90%.** This course grades *operation*, not artistry. The projects ask you to demonstrate that you can drive the machine, not that you made something beautiful. Grade yourself honestly: record your screen or your hands, watch it back cold, and ask "did I do that cleanly and without fumbling, or did I get lucky?"

## Course policies

- **Update firmware first.** A machine on old firmware is missing features you will be told to use. Non-negotiable, Day 0.
- **Back up weekly, and before anything risky.** The SP saves itself as you go, mistakes included, so the backup is your only undo and your only insurance. Lost work that had no backup is on you, and it is the top reason beginners quit.
- **Read the screen.** Most "I'm stuck" moments are answered by the text already on the display. Slow down and read it before reaching for help.
- **Do the machine task the same day you read the lecture.** Operation is muscle memory. Reading without doing does not transfer.
- **It is fine to fumble.** Speed comes later. The pass bar is "can you do it correctly," not "can you do it fast."
- **When wording differs from your firmware,** trust the printed labels and the reference manual; the underlying logic in this course still holds.
