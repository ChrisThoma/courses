# Answer Keys (MUS-40H)

Full keys for both quizzes and the final. Grade honestly and from memory. For written items, award partial credit where the core idea is right even if wording differs. For applied items, the standard is **clean operation, cold**: a fumbled or lucky pass is a partial, not a full. Mastery is 90%.

---

## Quiz 1 Key

**Part A (60 pts)**

1. **(6)** Sampler, four-bus effects processor, pattern sequencer, DJ mixer, USB audio interface. (~1 pt each, 6 for all five.)
2. **(6)** The machine's **internal storage** holds projects; a **project** holds 10 banks of 16 pads (plus patterns, tempo, settings); that is **160 pad slots**. (2 storage, 2 banks/pads, 2 total.)
3. **(6)** Press a **function button** (often with [SHIFT]), then **turn/press [VALUE]**, then **act on a pad**. (2 each.)
4. **(6)** Because the [CTRL] knobs do different jobs in different contexts; instead, **read the screen** to see what each knob does right now. (3 + 3.)
5. **(6)** A mic outputs a very weak signal needing lots of gain; a line source (phone/laptop) is much stronger. Mismatch: a mic in a line input is too quiet; a line source in a mic input is too hot/distorted. Match input to source, set level by meter. (3 concept + 3 failure modes.)
6. **(6)** Target: **loud but never clipping** (meter near the top on the loudest part, never slamming the ceiling). Wrong ways: too quiet (noisy/weak) or clipping (distorted, unrecoverable). (2 + 2 + 2.)
7. **(6)** **One-shot** = one press plays the whole sample (drum hits/one-offs); **gate** = plays only while held (rhythmic chops/stabs); **loop** = repeats until stopped (drum loops/beds). (2 each.)
8. **(6)** [REMAIN]/hold keeps a sample sounding **hands-free** so you can play other pads on top. Mono (cut its own sound): fast hi-hats, stuttering vocals. Poly: ringing piano, long pads. (2 + 2 + 2.)
9. **(6)** Arm [REC]; choose source; set level; choose dry or INPUT FX; start (manual or auto-trigger); stop to the destination pad. (1 each, 6 for all.)
10. **(6)** Skip-back grabs the **last several seconds the SP was already buffering** so you can capture something you missed. Auto-trigger **starts recording the instant sound crosses a threshold**, giving a clean front edge with no dead air. (3 + 3.)

**Part B (40 pts)**: full marks for clean, unhurried execution; deduct for hunting, wrong levels, hiccupping loops, or skipping the backup.
- **11 (10):** Opened the system menu, changed a setting, returned, no long hunting.
- **12 (10):** Healthy level set deliberately on the loudest part; no clip, not buried.
- **13 (10):** Dry sample via auto-trigger; loop is seamless (no audible seam).
- **14 (10):** Pad copied to another bank; backup run.

## Quiz 2 Key

**Part A (60 pts)**

1. **(6)** **BUS 1/2** are per-sample (insert) buses, applied only to pads routed to them; **BUS 3/4** are master buses that everything passes through, coloring the whole mix. (3 + 3.)
2. **(6)** A DRY pad **is** touched by a BUS 3 effect (everything flows through the master buses) but **is not** touched by a BUS 1 effect (it is not routed there). (3 + 3, with reasoning.)
3. **(6)** Vinyl Sim, Cassette, Lo-Fi: they add hiss, wow, and rolled-off highs, the classic SP grit/character. (name + effect.)
4. **(6)** The engine shares processing power across the buses, so there is a limit; when you hit it, **resample** to bake one effect into a sample and free the engine. (3 + 3.)
5. **(6)** INPUT FX is an effect stage on the input jacks only; it lets you **record a sound already colored** (the effect baked in). Use it when you want the effect committed at capture rather than added/removable later. (3 + 3.)
6. **(6)** Choose an empty pattern and arm [REC]; the pattern loops; play pads in time, layering parts each pass; stop. Tempo first because the **quantize grid is built from the BPM**. (4 steps + 2 tempo.)
7. **(6)** TR-REC places hits on a 16-step grid by hand (exact, methodical); real-time is played live in time (feel). Fix a wrong hit by **toggling that step off** in TR-REC, no re-recording. (3 + 3.)
8. **(6)** Quantize snaps hits to the nearest grid division. Light = tightens while keeping feel; full = machine-perfect but stiff. (3 + 3.)
9. **(6)** Resampling = **sampling the SP's own output to a new pad**. It lets you commit/print effects, glue a sequence into one pad, and free voices/pads. (3 sentence + 1 each for the three uses.)
10. **(6)** The DJFX looper live-captures a slice and repeats it, with the [CTRL] knob controlling slice size (stutters/builds). Keep a performance by **resampling** while you perform it. (3 + 3.)

**Part B (40 pts)**: full marks for correct routing demonstrated by ear and clean execution.
- **11 (10):** Insert effect on BUS 1 affects only the routed pad; proven by ear (route a second pad in/out to show the difference).
- **12 (10):** Master effect on BUS 3 audibly colors the whole mix.
- **13 (10):** One-bar TR-REC pattern, a live overdub layered on, take quantized.
- **14 (10):** Result resampled to a single pad and played back.

## Final Exam Key

**Part A (55 pts)**

1. **(5)** Pad → its pad settings (pitch, mode, volume, pan) → **BUS 1/2** (per-sample insert FX) → **BUS 3/4** (master FX on the whole mix) → LINE OUT/PHONES. (Name all four buses and the two roles.)
2. **(5)** Habit: **the weekly backup** (the SP itself saves as you go, so the backup is the only undo and insurance). Corruption: **yanking power (or pulling the SD card) while the machine is writing.** (3 + 2.)
3. **(5)** "On everything" = the effect is on a **master bus (BUS 3/4)**; "does nothing" = the pad is **not routed to that effect's bus**. (2.5 each.)
4. **(5)** USB audio carries the actual sound (streams in/out as an interface); USB MIDI carries performance/timing messages (notes, clock), not sound. The cable also carries **power**. (2 + 2 + 1.)
5. **(5)** Class-compliant = the computer recognizes the SP as an audio/MIDI device with **no driver to install**; plug in and it appears. (5.)
6. **(5)** Clock sync synchronizes **tempo, measure, and beat**; a setup has exactly **one clock leader**, everything else follows. (3 + 2.)
7. **(5)** The **USB audio routing setting in the Utility menu** (exact label varies by firmware); set it to **MIX OUT** to record the full post-effects mix. (3 + 2.)
8. **(5)** Enter DJ mode by pressing **[D/I] and [E/J] together**; [CTRL 1] = CH1 level, [CTRL 2] = CH2 level, [CTRL 3] = cue mix for headphone monitoring. (2 + 3.)
9. **(5)** Either **resample the beat to one pad and export that sample**, or **record the SP into a DAW over USB**. (2.5 each.)
10. **(5)** Start clean (load or create a project) → get sound in (level, auto-trigger) → edit (trim/loop/pitch) → shape (bus FX) → sequence (TR-REC + overdub, quantize) → commit (resample) → connect/export (DAW or export, backed up). (~0.7 each; 5 for the full ordered list.)
11. **(5)** Any three of: no sound → wrong bank/empty/level/output → check those; effect does nothing → not routed to that bus → route it; effect on everything → it is on a master bus → move to BUS 1/2; can't add more FX → processing limit → resample; DAW silent → USB routing/input → set MIX OUT and select SP; won't sync → leader/follower or channel → set one leader; lost work → deleted or overwrote it with no backup → restore from backup. (~1.7 each.)

**Part B (45 pts)**: grade strictly; this is the real exam. Deduct for hesitation, undo, guessing, manual use.
- **12 (10):** Clean sound in at a healthy level, edited onto a pad.
- **13 (10):** One insert + one master effect routed correctly; short pattern sequenced over it.
- **14 (10):** Result resampled to one pad.
- **15 (10):** SP recorded into a DAW over USB **or** a sample exported to the computer; backup current.
- **16 (5):** A fault correctly diagnosed aloud (symptom → cause) and fixed on the machine.

### Grading notes

- **Partial credit (written):** award it generously where the core operating idea is right; the wording on the SP varies by firmware and that is expected.
- **Applied parts are weighted to matter.** A student who aces the writing but fumbles the doing has not mastered operation. The reverse, shaky on wording but smooth on the machine, is closer to the goal; nudge them to firm up the model, but recognize the hands are most of the grade.
- **90% threshold:** below it, identify the weakest domain (use the [RUBRIC.md](RUBRIC.md) and [DRILLS.md](DRILLS.md)), drill it, and re-test that section. Operation sets in with reps, not with re-reading.
